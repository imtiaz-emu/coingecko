# shrtn.io

A production-grade URL shortener built with Rails 7.2. Shorten any URL, track click analytics by country and day, and get background title fetching ‚Äî all running on Render's free tier.

---

## Stack

| Layer | Technology |
|---|---|
| Web | Rails 7.2, Puma |
| Database | PostgreSQL 16 |
| Cache / Queue | Redis 7, Sidekiq 7 |
| UI | Bootstrap 5.3 CDN, Chart.js 4.4 CDN |
| Background Jobs | ClickRecordJob, TitleFetchJob, GeoIpResolveJob |
| Security | Rack::Attack (rate limiting), CSP, SSRF guard |
| Deployment | Render.com (web + worker services) |

---

## Features

- Shorten any public URL with an auto-generated or custom slug (up to 15 chars)
- Click tracking with geo-resolution via MaxMind GeoLite2 (city/country)
- Analytics page: total clicks, unique countries, click-by-day chart, paginated history
- Async title fetching ‚Äî result page polls until the page title is resolved
- REST API: `POST /api/v1/short_urls`, `GET /api/v1/short_urls/:code`, `GET /api/v1/short_urls/:code/analytics`
- Sidekiq Web UI at `/sidekiq` (HTTP Basic Auth)
- SSRF protection, CSP headers, `robots.txt` disallowing all crawlers
- Rate limiting: 10 URL creations / min and 120 redirects / min per IP

## Local Development

### Prerequisites

- Ruby 3.3.6 (via rbenv or rvm)
- Docker (for PostgreSQL + Redis)

### Setup

```bash
git clone <repo-url> && cd shrtn
cp .env.example .env          # fill in credentials if needed
bundle install
bin/local_db.sh               # starts postgres:5433 + redis:6379 via Docker
bin/rails db:create db:migrate
bin/rails server
```

Visit `http://localhost:3000`.

### Running Tests

```bash
bundle exec rspec             # 87 examples, ~1s
```

### Linting

```bash
bundle exec rubocop
```

### Background Jobs

Start Sidekiq in a separate terminal:

```bash
bundle exec sidekiq -C config/sidekiq.yml
```

### GeoIP (optional)

Download [GeoLite2-City.mmdb](https://dev.maxmind.com/geoip/geolite2-free-geolocation-data) and place it at `db/GeoLite2-City.mmdb`. Without it, geo fields remain null ‚Äî the app degrades gracefully.

## API

### Create a short URL

```
POST /api/v1/short_urls
Content-Type: application/json

{ "short_url": { "target_url": "https://example.com", "short_code": "myslug" } }
```

Response `201 Created`:
```json
{ "short_code": "myslug", "short_url": "https://shrtn.io/myslug", "target_url": "https://example.com" }
```

### Redirect info

```
GET /api/v1/short_urls/:short_code
```

### Analytics

```
GET /api/v1/short_urls/:short_code/analytics?page=1&per_page=20
```

## Deployment (Render)

The repo includes a `render.yaml` Blueprint that provisions:

- `shrtn-web` ‚Äî Rails/Puma web service
- `shrtn-worker` ‚Äî Sidekiq worker service
- `shrtn-redis` ‚Äî Redis instance (free tier)
- `shrtn-db` ‚Äî PostgreSQL database (free tier)

### Steps

1. Push repo to GitHub
2. Render Dashboard ‚Üí **New ‚Üí Blueprint** ‚Üí select this repo
3. Set the following environment variables manually in the Render dashboard (marked `sync: false`):
   - `APP_HOST` ‚Äî your Render domain, e.g. `shrtn-web.onrender.com`
   - `SIDEKIQ_WEB_USERNAME` ‚Äî basic auth username for `/sidekiq`
   - `SIDEKIQ_WEB_PASSWORD` ‚Äî strong password
4. Trigger deploy ‚Äî migrations run automatically via `bin/render-build.sh`

### Keep-warm (free tier)

Render free services spin down after 15 min of inactivity (~30s cold start).  
Set up [UptimeRobot](https://uptimerobot.com) (free) to ping `https://<your-app>.onrender.com/health` every 5 minutes.

### Post-deploy Checklist

- [ ] `GET /health` ‚Üí `{"status":"ok"}`
- [ ] Submit a URL on home page ‚Üí short URL displayed
- [ ] Follow short URL ‚Üí redirects to target
- [ ] Wait ~10s ‚Üí title populates on result page
- [ ] `GET /:code/stats` ‚Üí analytics page with click count
- [ ] `GET /robots.txt` ‚Üí `Disallow: /`
- [ ] `GET /sidekiq` ‚Üí basic auth prompt
- [ ] Submit `http://192.168.1.1` ‚Üí blocked with error (SSRF guard)
- [ ] HTTPS enforced (Render provides TLS automatically)
